# Instructor-Guided Evaluation of Knowledge Components

This repository contains the dataset and evaluation code for the paper:

> Rafaella Sampaio de Alencar and Peter Brusilovsky. 2026. *Beyond Learning Curves:
> Instructor-Guided Evaluation of Knowledge Components.* Under review.

## Overview

This work introduces two complementary frameworks for evaluating the quality of
Knowledge Component (KC) annotations using instructor-defined course structure as
an evaluation signal — without requiring large volumes of student interaction data.

The two evaluation methods are:
- **Course Placement Evaluation**: tests whether KC annotations can reproduce
  instructor-defined activity placements within an ordered topic sequence.
- **Bundle Consistency Evaluation**: assesses whether KC annotations produce
  coherent and separable profiles for groups of thematically related activities.

Both methods are applied to six KC annotation conditions spanning expert-authored,
parser-based, and LLM-based approaches, on a real-world introductory Python
programming course.

instructor-guided-kc-evaluation/

│                                     # Educational-Ontology_Expert, AST_Parser,

│                                              # generated by

## Usage

### 1. Course Placement Evaluation

Run `course_placement_pipeline.ipynb` to generate placement results for each
annotation condition. This reads `annotations/course_annotations.csv` and writes
one output folder per condition under `outputs/`.

Then run `course_placement_analysis.ipynb` to compute summary metrics (accuracy,
Cohen's κ, placement rate), generate confusion matrices, and reproduce the figures
used in the paper.

### 2. Bundle Consistency Evaluation

Run `bundle_evaluation.ipynb` to compute within- and between-bundle similarity
scores and leave-one-out bundle assignment accuracy. This reads
`annotations/bundle_annotations.csv`.

## Annotation Conditions

| Label | Description |
|---|---|
| Syntax-Ontology Expert | Expert annotation using the Python ontology |
| Educational-Ontology Expert | Expert annotation using the Educational Python ontology |
| AST Parser | Automated annotation using the ACOS Python parser |
| Syntax-Ontology LLM | LLM annotation using CLAWS with the Python ontology |
| Educational-Ontology LLM | LLM annotation using CLAWS with the Educational Python ontology |
| Free-Form LLM | Automated annotation using KCGen-KT (no predefined ontology) |
